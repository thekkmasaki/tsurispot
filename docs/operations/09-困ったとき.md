# 09. 困ったとき（トラブル対応）

> 何か問題が起きたら、まずこの章。症状から逆引きで対処法を探せます。
> 大原則：自分でむやみにコマンドをいじらない。症状を正確に伝えて助けを求める方が早くて安全。

最終更新: 2026-07-07

---

## まず落ち着く：多くの問題は「サイトは無事」
- デプロイが失敗しても、本番サイトは古いまま動き続けます（壊れた版には切り替わらない）
- 自動投稿が失敗しても、サイト自体は無事
- 本当にサイトが止まるケースは稀

まず「本当にサイトが見えないのか」を、スマホなど別の回線で tsurispot.com を開いて確認しましょう。

---

## 症状別・逆引き対処表

### サイトが表示されない・重い
1. スマホの回線（Wi-Fi以外）でも開けないか確認（自分のPCだけの問題かも）
2. AWS → App Runner（東京）で、サービスが「Running」か確認
3. Cloudflare（dash.cloudflare.com）で障害が出ていないか
4. 直前にデプロイした？ → その変更が原因かも。04章の「戻し方」でrevertを依頼
5. 解決しなければ開発者/Claudeに「サイトが○○（症状）、○時頃から」と伝える

### 広告が表示されない・収益が急落
1. AdSense → ポリシーセンターを確認（違反・警告があれば最優先で対処）
2. GA4でPVが落ちていないか（アクセス減なら収益減は当然）
3. 02章の「収益が落ちたとき」を参照
4. 季節要因（1〜2月は広告単価が下がりがち）の可能性も

### PVが急に増えた／減った
- 急増したのに収益が増えない → ボットを疑う（05章・05b）
- 急減 → ①Google検索の変動 ②インデックス減 ③サイト障害（03章）

### SNS自動投稿が止まった
1. GitHub → Actionsタブで「X 自動投稿」の赤い失敗を確認
2. Discordの通知も確認
3. よくある原因：APIキー/トークンの失効 → 各サービスで再発行しGitHub Secretsを更新
4. 06章参照

### Instagramに投稿できない
- トークン（鍵）が60日で切れた可能性が最有力（06章）
- トークンの再取得・更新が必要（技術作業）

### デプロイが失敗する（赤い失敗）
1. GitHub → Actions → 失敗した実行 → ログの赤い部分
2. よくある原因：型エラー／テスト失敗／ビルドが重すぎる
3. 本番サイトは無事なので慌てない
4. ログをコピーして開発者/Claudeへ

### ブログが更新できない
- microCMS（tsurispot.microcms.io）にログインできるか
- APIキーの問題なら07章のSecrets確認

### AWSの請求が急に高い
1. AWS → Billing で、どのサービスが増えたか
2. 通常は月$121前後（App Runner $88＋DynamoDB $20＋その他）
3. 急増の心当たり：ボットの大量アクセス／CloudFrontの無駄課金（07章）

## 緊急度の判断
| レベル | 状況 | 対応 |
|---|---|---|
| 最優先 | サイトが完全に見えない／広告アカウント停止警告 | すぐ開発者/Claudeへ。症状・発生時刻を正確に |
| 注意 | 一部機能が変・SNS投稿停止・収益ダウン | その日のうちに確認。上の逆引き表で対処 |
| 様子見 | IndexNow失敗・軽微な表示崩れ | 週次チェックのときに対応でOK |

---

## 助けを求めるときの「良い伝え方」
この4点を伝えると解決が早いです：
1. 症状：何が、どうなっているか（例「スポットページだけ広告が出ない」）
2. いつから：気づいた時刻・きっかけ（例「今日の昼のデプロイのあと」）
3. 範囲：全ページか一部か、自分だけか他の人もか
4. やったこと：すでに試したこと（例「スーパーリロードした」）

良い例：「スポット詳細ページで、今日15時のデプロイ後から広告が表示されない。トップは正常。スマホでも同じ」

## 用語ミニ辞典
- revert：変更を取り消して元の状態に戻すこと。
- 500エラー：サーバー側の障害を示すエラー画面。
- スーパーリロード：キャッシュを無視して再読み込み（Windows: Ctrl+Shift+R）。
- トークン失効：SNS投稿用の鍵の期限切れ。再取得が必要。
