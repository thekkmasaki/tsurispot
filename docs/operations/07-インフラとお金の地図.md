## サイトが動いている場所（インフラ地図）
```
ユーザー
  ↓
Cloudflare … 門番＆高速化（無料）
  ↓
AWS App Runner … サイト本体を動かすサーバー（月$88）
  ↓ 使う
DynamoDB … ページのキャッシュ保存（月$20）
Upstash Redis … オンライン人数・釣果投稿（ほぼ無料）
S3 … Instagram動画の一時置き場（少額）
```

| サービス | 役割 | 管理画面 | 月額 |
|---|---|---|---|
| AWS App Runner | サイト本体のサーバー（2vCPU/4GB、東京） | console.aws.amazon.com → App Runner | $88 |
| AWS DynamoDB | ページのキャッシュ保存 | 同上 → DynamoDB | $20 |
| AWS ECR | サイトの完成品保管庫 | 同上 → ECR | 少額 |
| AWS Route 53 | ドメイン管理 | 同上 → Route 53 | 約$0.5 |
| AWS S3 | Instagram動画の一時置き場 | 同上 → S3 | 少額 |
| Cloudflare | 門番・高速化・ボット対策 | dash.cloudflare.com | 無料 |
| Upstash Redis | リアルタイム機能 | console.upstash.com | ほぼ無料 |

> CloudFrontについて：古いメモに「CloudFront」とあるが、実際のCDNはCloudflare。コードにCloudFrontの痕跡はなし。念のためAWS請求にCloudFront項目がないか一度確認しておくと安心（あれば無駄な課金）。

## 費用を下げる余地（将来）
AWSの$88（App Runner）が最大の出費。原因は大量URL（約18,760）をボットや検索エンジンが巡回しサーバーが高負荷なこと。対策の順番：①キャッシュ保持を長く ②不要な大量ページのクロールを減らす ③表示を軽く → そのあとサーバーを 2vCPU→1vCPU に縮小すれば費用減。※技術作業。今すぐ不要だが「App Runnerが一番の出費」と覚えておけばOK。

## 用語ミニ辞典
- インフラ：サイトを動かす裏側の設備。
- App Runner：AWSの、サイトを動かすサーバー。一番の出費。
- DynamoDB / Redis：データの保管庫（用途が違う2種類）。
- CDN：世界中にコピーを置いて高速表示する仕組み。ここではCloudflare。
- vCPU：サーバーの処理能力の単位。多いほど速いが高い。
