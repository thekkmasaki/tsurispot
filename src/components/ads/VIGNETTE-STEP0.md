# STEP 0: AdSense ビネット広告（CLS安全・PV据え置きで増収）

広告収益を **PVを増やさず** 引き上げるための第一手。`ad-unit.tsx` のコードはほぼ変えず、
**AdSense管理画面で「ビネット」だけを有効化**する。ここはその設定手順・根拠・運用を記録する。

## なぜビネットだけか

- このサイトのRPMは趣味ジャンルのAdSense相場の上限付近。**広告枠を増やすのは逆効果**
  （密度30%超で viewability 低下 → CPM下落がインプレ増を上回り総収益が反転低下＝inverted-U）。
- 過去に **ATFヘッダー広告でCLS悪化 → 検索順位下落** の実害があり削除済み（`layout.tsx` 冒頭コメント参照）。
  SEOが事業の生命線のため、CLSを1ミリも悪化させない施策しか採らない。
- **ビネット（ページ遷移時の全画面オーバーレイ）はオーバーレイ表示**。in-page要素を押し下げないので
  **構造的に CLS=0**。安全に足せる唯一の高RPMフォーマット（想定RPM寄与 +5〜15%）。

## 管理画面の設定手順（AdSense）

1. AdSense → **広告** → **サイトごと** → `tsurispot.com` → **編集**（自動広告）
2. **自動広告を ON**
3. **「ページ内フォーマット」を全て OFF**
   - 記事内・関連コンテンツ等。既存の手動広告枠（`ad-unit.tsx` の各コンポーネント）と二重化し、
     かつ自動挿入は領域予約されずCLSを悪化させるため。
4. **「オーバーレイ フォーマット」→ アンカー広告を OFF**
   - 手動の `MobileStickyAd`（領域予約・dismissボタン付き）と二重化するため。
5. **「ビネット（ページ間広告）」のみ ON**
6. **フリークエンシー上限を設定**（例：1ユーザー / 60分に1回）。UX過剰・離脱増を防ぐ。
7. （任意）有料店舗ページなど広告非表示にしたいURL（コード側で `body[data-no-ads="true"]`）は、
   管理画面の **URL除外** に追加。※ビネット(自動広告)は手動枠の `useAdsSuppressed()` とは別系統のため、
   コードの抑制では止まらない。除外は管理画面側で行う。

## コード側の状態（変更不要）

- `adsbygoogle.js` は `src/app/layout.tsx` で全ページ `lazyOnload` 読込済み → ビネットに必要なタグは充足。
- 手動広告枠は `src/components/ads/ad-unit.tsx` に集約。全枠 `min-h-[250px]`（ヘッダー90px / サイドレール600px）で
  **領域予約済み**＝CLS対策は実装済み。今回これらは触らない。

## やらないこと

- ❌ インページ自動広告の有効化（CLS悪化＝過去の順位下落の再来）
- ❌ ATFヘッダー広告の再有効化
- ❌ 広告枠数の増加（inverted-Uで逆効果）

## 検証・モニタリング

- **ベースライン**：適用前のCLS（GA4 web-vitals、`src/components/web-vitals-reporter.tsx` 経由で計測）と
  RPM（AdSense直近28日）を記録。
- **適用後**：
  - RPM：手取り（手数料控除後）で 適用前28日 vs 後28日 を比較（ビネット想定 +5〜15%）。
  - CLS：GA4 web-vitals で悪化が無いことを確認（オーバーレイなので理論上ほぼ不変）。
  - 直帰率・INP：主要ページ（スポット詳細・ホーム）で離脱増が無いか監視。

## ロールバック

- **管理画面で「ビネット」を OFF にするだけ**（デプロイ不要・即時）。コード変更を戻す必要はない。

デザイン監査（2026-07）で本文書の方針と本番の実挙動に乖離が観測されたため、Phase 0 として以下を実施する。

### 1. 観測された乖離

- 本番のモバイル表示で **ヘッダー上に高さ約120pxの自動広告帯** ＋ **アンカー広告** を観測。
- 本文書の方針は「ページ内フォーマット全OFF / アンカーOFF / ビネットのみON」であり、
  **管理画面の設定が方針とズレている（または後から自動で有効化された）疑いが強い**。
- ページ内自動挿入は領域予約されずCLSを悪化させる＝過去のATF広告による順位下落の再来リスク。
  アンカーは手動 `MobileStickyAd` と二重化しポリシー/UXリスクになる。

### 2. AdSense管理画面での確認・復帰手順

1. AdSense → **広告** → **サマリー** → **サイトごと** → `tsurispot.com` → **編集**（自動広告）
2. **広告掲載の自動化** の設定を以下の状態に戻す:
   - **ページ内フォーマット: 全て OFF**（記事内・バナー・関連コンテンツ等）
   - **アンカー広告: OFF**
   - **ビネット: ON**（フリークエンシー上限も維持されているか確認）
   - **広告読み込み量: 下げる**（スライダーを低め設定）
3. 保存（反映まで最大1時間程度かかる場合あり）。

### 3. 設定変更後の確認方法

- **シークレットウィンドウ**を開き、DevTools のデバイスエミュレーションで**スマホUA**にして
  `https://tsurispot.com/`・スポット詳細・魚種ページを数ページ閲覧。
- ヘッダー上の自動広告帯・アンカー広告が**出ないこと**、ページ遷移時のビネットは出ることを確認。
- 数日後に GA4 web-vitals で CLS が改善（悪化していないこと）を確認。
